## Methodology (how each metric was calculated)

- **Tool selection accuracy** — of 5 tests, the fraction where the agent's actual tool-call sequence (as a multiset, order-independent) exactly matched the expected tool(s) for that test.
- **Argument accuracy** — micro-averaged across every individual argument check performed (4/5 checks matched): each expected tool call's arguments were checked field-by-field, where free-text fields (title, description, context, meeting_text) only require a non-empty value and controlled-vocabulary fields (priority, status, category) require an exact match.
- **Task completion rate** — fraction of tests that passed end-to-end (correct tool(s), correct arguments, correct final status, and — for approval tests — correct behavior after the approve/reject/edit decision was submitted).
- **Approval compliance** — of 2 tests involving a sensitive tool (`update_task`, `complete_task`, `convert_meeting_notes_to_tasks`), the fraction where the agent correctly paused with `status="awaiting_approval"` before that tool executed, with no sensitive tool ever executing without a prior human decision.
- **Invalid action rate** — fraction of all tests where the agent took an action it should not have: executing a sensitive tool without an approval pause, calling a tool on a direct-response prompt that needed none, or guessing a target (e.g. resolving an ambiguous "the third one" reference) instead of asking for clarification.
- **Average response time** — mean wall-clock latency, in milliseconds, of every `POST /api/chat` and `POST /api/approvals/{id}` call made during the run, timed client-side by the evaluation harness.
- **Recovery rate** — of 1 failure/edge-case tests engineered to trigger an error condition (not-found ID, blank input, oversized input), the fraction where the agent degraded gracefully — returning a structured `ToolResult(success=false, error=...)` and a coherent final reply — instead of an unhandled exception or a raw `status="error"` crash.
